# IMPLEMENTATION PLAN – VESSEL MONITORING ALERT SYSTEM (FOR ANTIGRAVITY)

---

## 1. Objective

Upgrade the current Vessel Monitoring System (VMS) from **anomaly detection** to a **full alerting and maritime intelligence platform**.

This implementation plan guides Antigravity to build:

* Alert Engine
* Real-time Detection
* Notification System
* Geo-fencing Intelligence
* Scalable Architecture

---

## 2. Current System Baseline

### Existing Components

* `vessels`, `vessel_tracks`
* `anomaly_rules`, `anomaly_events`
* Detection via Edge Function (cron 5 min)
* API endpoints for anomaly management

### Limitations

* No alert aggregation
* No notification delivery
* Not real-time
* No geo-intelligence
* No scalability layer

## 4. Phase 1 – Alert Engine (CORE)

### 4.1 Database Schema

```sql
CREATE TABLE alerts (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  vessel_id uuid,
  rule_id uuid,
  severity text,
  status text DEFAULT 'open',
  event_count int DEFAULT 1,
  first_event_id uuid,
  last_event_id uuid,
  assigned_to uuid,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);
```

### 4.2 Logic

* Group anomaly_events by:

  * vessel_id
  * rule_id
  * time window (e.g. 15 minutes)
* Avoid duplicate alerts
* Increment `event_count`

### 4.3 Alert Lifecycle

```
open → acknowledged → resolved → dismissed
```

### 4.4 Tasks for Antigravity

* Create Alert Service (TypeScript)
* Build aggregation logic
* Implement update/merge strategy
* Expose APIs:

  * GET /alerts
  * PATCH /alerts/:id

### 5.1 Replace Cron with Event-driven

### 5.3 Detection Functions

* checkDarkVessel
* checkSpoofing
* checkSpeedAnomaly
* checkLoitering

### 5.4 Requirements

* Stateless functions
* Idempotent execution

## 6. Phase 3 – Notification System

### 6.1 Database Schema

```sql
CREATE TABLE notification_channels (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid,
  type text,
  config jsonb,
  is_active boolean DEFAULT true
);

CREATE TABLE notification_logs (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  alert_id uuid,
  channel text,
  status text,
  sent_at timestamptz DEFAULT now()
);
```
